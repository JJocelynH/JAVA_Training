7. Autowire using spring boot
Because both `Laptop` and `Dev` are annotated with `@Component`, Spring automatically registers them as Beans in the IoC container.

`@Autowired` tells Spring: "I need a `Laptop` bean — find one in the container and inject it for me."

The flow:

1. Spring starts up and scans for all `@Component` classes
2. Finds `Laptop` → creates a Laptop bean and puts it in the container
3. Finds `Dev` → creates a Dev bean, sees `@Autowired`, automatically injects the `Laptop` bean

So you never need to do `new Laptop()` manually — Spring handles object creation and injection for you.

constructor injection doesn't need @Autowire, setter and field doesn't need one
@Autowire looking for type

when you have many different beans in the jvm and competing for autowire, you may delete one @component or use @primary to define which to autowired first
or use @qualifier behind @autowire, to choose which is going to be autowired.
