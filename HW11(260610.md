1. Why spring boots? pros and cons
	**Pros:**
	
	- **Auto-configuration** — automatically configures your app based on the dependencies you added, no need for verbose XML or manual Bean setup
	- **Embedded server** — comes with built-in Tomcat, so you just run `java -jar`, no separate server deployment needed
	- **Starter dependencies** — one starter manages a group of related dependencies with compatible versions
	- **Production-ready** — built-in Actuator for health checks and monitoring out of the box
	- **Rich ecosystem** — seamlessly integrates with Spring Security, Spring Data, Spring Cloud
	
	**Cons:**
	
	- **Magic black box** — auto-configuration happens behind the scenes, making it harder to debug when things go wrong
	- **Large JAR size** — packages many dependencies you might not actually use
	- **Slower startup** — compared to lightweight frameworks like Micronaut or Quarkus
	- **Higher memory footprint** — not ideal for resource-constrained environments
