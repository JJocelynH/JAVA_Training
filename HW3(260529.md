1. thread lifecycle, how does thread transfer from one state to another

new, runnable, blocked, waiting, timed waiting and terminated. 
A thread starts in the NEW state when created but not yet started. Calling start() moves it to RUNNABLE, where it's either running or ready to run waiting for CPU time.
From RUNNABLE, a thread can go into one of three waiting states. It enters BLOCKED when it tries to acquire a synchronized lock that another thread is holding — it stays there until the lock is released. It enters WAITING when it calls wait(), join(), or LockSupport.park(), and it can only wake up when another thread explicitly calls notify() or notifyAll(). It enters TIMED_WAITING when it calls sleep(ms), wait(ms), or join(ms) — it wakes up automatically when the timeout expires, or earlier if notified.
Once the run() method finishes or throws an unhandled exception, the thread moves to TERMINATED and cannot be restarted.
The key distinction to remember: BLOCKED is always about waiting for a lock, while WAITING and TIMED_WAITING are about waiting for a signal from another thread.
