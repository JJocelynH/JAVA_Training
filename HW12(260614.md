


1.completablefuture / multithreading in spring framework
Spring handles async through @Async and 
CompletableFuture, with separate thread pools for primary and auxiliary tasks.

I annotate a configuration class with @EnableAsync and implement AsyncConfigurer to define a custom ThreadPoolTaskExecutor with core and max pool sizes. Service methods annotated with @Async return CompletableFuture and run in a background thread. The key design is separating tasks: primary tasks like order processing run on the main thread pool, auxiliary tasks like email and SMS notifications run on a separate auxiliary pool so they never block core operations. For thread safety, I use AtomicInteger or AtomicReference for lock-free operations, ConcurrentHashMap for thread-safe collections, and ReentrantLock when I need more control over locking.

2.what annotations we use to configure customized actuator
Use @Endpoint on the class and @ReadOperation, @WriteOperation, or @DeleteOperation on the methods.

@Endpoint takes an id which becomes the URL path under slash actuator slash. @ReadOperation maps to GET, @WriteOperation to POST, @DeleteOperation to DELETE. Spring Boot registers it automatically.

3.how Spring IOC work
IOC means the ApplicationContext container takes over creating and managing objects. At startup it scans annotations, creates bean definitions, instantiates beans, and performs dependency injection automatically.

If circular dependencies occur — bean A depends on B and B depends on A — use @Lazy on one of them to defer its creation until it's actually needed.

4. all annotations and injection and bean types
Bean declaration: @SpringBootApplication combines @EnableAutoConfiguration, @Configuration, and @ComponentScan. Under scanning: @RestController, @Service, @Repository, @Component. @Bean inside @Configuration for third-party classes.

Injection: @Autowired triggers injection. Three types: constructor (recommended), field (hard to test), setter (rarely used). Resolve conflicts with @Qualifier by name or @Primary as default. Scopes via @Scope: singleton (default), prototype, request, session, application.

5. Could AOP apply to the private method?
No. Spring AOP uses proxies which can only intercept public method calls from outside the class. Private methods are invisible to the proxy.

Even public methods bypass AOP if called with this dot inside the same class. The call must go through the Spring-managed proxy — meaning from an injected bean from outside.

6.how can you use profile
Four ways to activate a profile: set spring.profiles.active in application.properties, pass it as a command line argument, set it as an environment variable on the server, or use @ActiveProfiles on a test class.

Command line example: java dash jar app.jar dash dash spring.profiles.active equals qa. Environment variable on Linux: export SPRING_PROFILES_ACTIVE equals QA. The environment variable approach is most secure for production because configuration is separated from code.

7.how do you define profile 
Two ways: create separate configuration files per environment, or use @Profile on a bean.

Files are named application dash the environment name in properties or YAML format, like application-dev.properties or application-qa.yml. @Profile on a bean or configuration class ensures that bean only loads when the specified profile is active. This enables environment segregation across dev, QA, UAT, staging, and production.

8. why AOP
Two reasons: to enhance functionality by extracting cross-cutting logic out of serial business flow, and to avoid duplicate code patterns across the application.

Without AOP you'd write the same logging or security check in every method. AOP defines it once in an Aspect class and applies it everywhere automatically. Common use cases are logging, security checks, and performance timers.

9.what java version can we use with spring boot 3
Spring Boot 3 requires a minimum of Java 17. It also supports Java 21 which adds Virtual Threads.

The most critical change from Boot 2 is the javax to jakarta package rename — all imports must be updated. Java 21 Virtual Threads improve concurrent performance and reduce the need for reactive WebFlux in many cases.

10.Dependency injection types
Three types: constructor injection (recommended), field injection, and setter injection.

Constructor injection prevents NullPointerException because all dependencies are required at object creation time, and it makes unit testing easy. If circular dependencies occur, use @Lazy to break the cycle. Field injection is convenient but hard to test. Setter injection makes dependencies optional leading to incomplete object states, so it's rarely used.

11.how you implement exception in springboot
Two approaches: @RestControllerAdvice with @ExceptionHandler for returning clean HTTP responses, and AOP with @AfterThrowing for centralized logging.

Java has two exception types: runtime exceptions (unchecked) and compile-time exceptions (checked). @RestControllerAdvice catches exceptions globally and returns ResponseEntity with the right status code. AOP's @AfterThrowing logs the exception details. The two work together — AOP logs, @RestControllerAdvice responds.

12.what spring boot version you used
I used Spring Boot 3, which requires Java 17 and migrated all packages from javax to jakarta.

For example javax.persistence.Entity becomes jakarta.persistence.Entity. This is the biggest breaking change when upgrading from Boot 2. Boot 3 is also built on Spring Framework 6 and Jakarta EE 10.

13.what annotations you used in spring
IOC: @SpringBootApplication, @RestController, @Service, @Repository, @Component, @Bean with @Configuration, @Scope.

DI: @Autowired, @Qualifier, @Primary. AOP: @Aspect, @Pointcut, @Around, @Before, @After, @AfterThrowing, @AfterReturning. REST: @RequestMapping, @GetMapping, @PostMapping, @PutMapping, @DeleteMapping, @PathVariable, @RequestParam, @RequestHeader, @RequestBody, ResponseEntity. Exception: @RestControllerAdvice, @ExceptionHandler. Validation: @Valid, @NotNull, @NotBlank, @Email, @Size.

14.Type vs name
Spring injects by type first. If multiple beans of the same type exist, it falls back to by name. Use @Qualifier to explicitly specify which bean by name, or @Primary to set a global default.

If neither resolves the conflict, Spring throws NoUniqueBeanDefinitionException at startup. @Qualifier is placed at the injection point, @Primary is placed on the bean definition.

15.what is enable auto configuration
@EnableAutoConfiguration tells Spring Boot to automatically configure the application based on what dependencies are on the classpath.

For example if JPA is present it sets up a datasource automatically. It's already included inside @SpringBootApplication so you don't add it separately.

16.how to stop auto configuration in spring boot
Use the exclude attribute in @SpringBootApplication to exclude specific auto-configuration classes, or set spring.autoconfigure.exclude in application.properties.

For example to prevent Spring from auto-configuring the datasource, exclude DataSourceAutoConfiguration in the annotation. Useful when you want to manually configure something instead of relying on the default auto-configuration.
